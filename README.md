# LangGraph Linear Workflow

A beginner-friendly demonstration of building a **linear multi-node workflow** using [LangGraph](https://github.com/langchain-ai/langgraph). The project shows how to define shared state, wire up nodes into a graph, and invoke it end-to-end.

---

## What is LangGraph?

LangGraph is a library built on top of LangChain that lets you model agentic workflows as a **directed graph**. Each node in the graph is a Python function that reads from and writes to a shared state object. Edges define the execution order.

## linear.py — Detailed Explanation

### 1. Imports

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, END
from langchain_core.messages import HumanMessage, AIMessage
import operator
import re
```

| Import | Purpose |
|--------|---------|
| `TypedDict` | Defines the shape of the shared state as a typed dictionary |
| `Annotated` | Attaches metadata (a reducer function) to a field |
| `StateGraph` | The main LangGraph class for building graphs |
| `END` | A sentinel that marks the terminal node of the graph |
| `HumanMessage / AIMessage` | LangChain message types for conversation history |
| `operator` | Provides `operator.add` used as the messages reducer |
| `re` | Used for whole-word keyword matching |

---

### 2. State Schema — `AgentState`

```python
class AgentState(TypedDict):
    messages: Annotated[list, operator.add]
    user_input: str
    processing_step: str
    result: str
```

`AgentState` is the **single source of truth** passed between every node. Each field has a specific role:

| Field | Type | Description |
|-------|------|-------------|
| `messages` | `Annotated[list, operator.add]` | Accumulates conversation history across nodes. `operator.add` is the **reducer** — instead of overwriting, each node's returned list is appended to the existing one. |
| `user_input` | `str` | The original query from the user. Set once at invocation and never mutated by nodes. |
| `processing_step` | `str` | Tracks which stage the workflow is at (`initialized` → `input_received` → `analyzed` → `processed` → `completed`). |
| `result` | `str` | The final response generated by `process_node`. Read by `output_node` to produce the closing message. |

> **Key concept — reducers:** By default, LangGraph overwrites a state field when a node returns a new value. Using `Annotated[list, operator.add]` tells LangGraph to *merge* (append) instead of overwrite for the `messages` field. This is how conversation history accumulates across nodes.

---

### 3. Node Functions

The workflow has four nodes, each a plain Python function with the signature `(state: AgentState) -> dict`.

#### Node 1 — `input_node`

```python
def input_node(state: AgentState) -> AgentState:
    return {
        "messages": [HumanMessage(content=state["user_input"])],
        "processing_step": "input_received"
    }
```

- Reads `user_input` from state
- Wraps it in a `HumanMessage` and adds it to `messages`
- Sets `processing_step` to `"input_received"`

#### Node 2 — `analyze_node`

```python
def analyze_node(state: AgentState) -> AgentState:
    user_text = state["user_input"].lower()
    words = re.findall(r'\b\w+\b', user_text)

    if "weather" in user_text:
        analysis = "Weather-related query detected"
    elif "hello" in words or "hi" in words:
        analysis = "Greeting detected"
    else:
        analysis = "General query detected"

    return {
        "messages": [AIMessage(content=f"Analysis: {analysis}")],
        "processing_step": "analyzed"
    }
```

- Lowercases the input and extracts individual words using `re.findall(r'\b\w+\b', ...)`
- Categorises the input as **Weather**, **Greeting**, or **General**
- `re.findall` is used instead of `"hi" in user_text` to avoid false positives from substrings (e.g. `"somet`**hi**`ng"` would incorrectly match a plain substring check)
- Appends an `AIMessage` with the analysis result to `messages`

#### Node 3 — `process_node`

```python
def process_node(state: AgentState) -> AgentState:
    user_text = state["user_input"].lower()
    words = re.findall(r'\b\w+\b', user_text)

    if "weather" in user_text:
        response = "I would check the weather API for current conditions."
    elif "hello" in words or "hi" in words:
        response = "Hello! How can I help you today?"
    else:
        response = f"I received your message: '{state['user_input']}'"

    return {
        "messages": [AIMessage(content=response)],
        "processing_step": "processed",
        "result": response
    }
```

- Applies the same keyword logic as `analyze_node` to generate an actual response
- Writes the response to both `messages` and the `result` field
- `result` is the primary output consumed downstream

#### Node 4 — `output_node`

```python
def output_node(state: AgentState) -> AgentState:
    final_message = f"Final Response {state['result']}"
    return {
        "messages": [AIMessage(content=final_message)],
        "processing_step": "completed"
    }
```

- Reads `result` from state (written by `process_node`)
- Wraps it in a final `AIMessage` and marks the workflow as `"completed"`

---

### 4. Graph Construction

```python
workflow = StateGraph(AgentState)

workflow.add_node("input",   input_node)
workflow.add_node("analyze", analyze_node)
workflow.add_node("process", process_node)
workflow.add_node("output",  output_node)

workflow.set_entry_point("input")

workflow.add_edge("input",   "analyze")
workflow.add_edge("analyze", "process")
workflow.add_edge("process", "output")
workflow.add_edge("output",  END)

app = workflow.compile()
```

This builds and compiles the graph into `app` at **import time**. The execution flow is a strict linear pipeline:

```
START --> input_node --> analyze_node --> process_node --> output_node --> END
```

`workflow.compile()` validates the graph (entry point set, no dangling edges) and returns a runnable `app` object.

---

### 5. Running the Workflow

```python
def run_workflow(user_input: str):
    initial_state = {
        "messages": [],
        "user_input": user_input,
        "processing_step": "initialized",
        "result": ""
    }
    final_state = app.invoke(initial_state)
    return final_state
```

- Constructs the initial state with empty `messages` and `result`
- Calls `app.invoke()` which drives the state through all four nodes sequentially
- Returns the final state after `output_node` completes
- `final_state["messages"]` will contain **4 messages** (one appended per node)
